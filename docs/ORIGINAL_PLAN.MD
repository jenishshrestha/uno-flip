# UNO Flip - Local Multiplayer Card Game

## Context

Office team plays double-sided UNO (UNO Flip) face-to-face, but the rules are confusing in person. Building a local network multiplayer game so 4-10 players in the same room can join instantly from their phones and play with rules enforced automatically.

**New repo** вҖ” completely separate from heubert-monolith-starter.

---

## Tech Stack

| Layer | Technology | Reasoning |
|-------|-----------|-----------|
| Runtime | Node.js + TypeScript | Shared types between server & client |
| Server framework | Express | Serves static files + API |
| Real-time | Socket.IO | Rooms, reconnection, namespaces built-in |
| Client | React + Vite | Fast dev, fast builds, small bundle |
| Styling | Tailwind CSS v4 | Mobile-first, utility classes |
| Card animations | CSS 3D transforms | Hardware-accelerated flip animations |
| Game logic | State machine (custom) | Authoritative server, prevents cheating |
| QR code | `qrcode` package | Display join URL on host screen |
| Monorepo | Turborepo or npm workspaces | Shared types between server & client |

## Implementation Phases

### Phase 1: Project Setup & Shared Types
> Foundation вҖ” get the monorepo running with shared types

- [ ] Initialize repo with npm workspaces
- [ ] Setup TypeScript config (base + per-package)
- [ ] Define shared types:
  - `CardSide` вҖ” { color, value, type (number|action|wild) }
  - `Card` вҖ” { id, lightSide: CardSide, darkSide: CardSide }
  - `ActiveSide` вҖ” 'light' | 'dark'
  - `Player` вҖ” { id, name, handCount, isUno, connected }
  - `GameState` вҖ” { activeSide, discardTop, currentPlayer, direction, players, phase }
  - `PlayerHand` вҖ” (private, only sent to owning player)
- [ ] Define all 112 cards in deck constant (light + dark pairings)
- [ ] Define Socket.IO event types (strongly typed):
  - Client вҶ’ Server: `CREATE_ROOM`, `JOIN_ROOM`, `PLAY_CARD`, `DRAW_CARD`, `CALL_UNO`, `CATCH_UNO`, `SELECT_COLOR`
  - Server вҶ’ Client: `ROOM_CREATED`, `PLAYER_JOINED`, `GAME_STATE`, `HAND_UPDATE`, `CARD_PLAYED`, `FLIP_EVENT`, `ROUND_OVER`, `GAME_OVER`, `ERROR`

### Phase 2: Server вҖ” Room System
> Players can create and join rooms

- [ ] Express server serving static client build
- [ ] Socket.IO server with typed events
- [ ] Room manager:
  - Create room вҶ’ generates 4-char room code (e.g., `ABCD`)
  - Join room вҶ’ validates code, checks player count (4-10)
  - Player names (no auth, just pick a name)
  - Host designation (first player = host, can start game)
  - Handle disconnection/reconnection (Socket.IO built-in + rejoin by player ID)
- [ ] LAN IP detection + QR code generation (terminal + web dashboard for host)

### Phase 3: Server вҖ” Game Engine (Core Loop)
> The heart of the game вҖ” state machine + rules enforcement

- [ ] **Deck management:**
  - Shuffle algorithm (Fisher-Yates)
  - Deal 7 cards to each player
  - Draw pile + discard pile tracking
  - Reshuffle discard into draw when empty
  - Flip entire deck/hands when FLIP card played

- [ ] **Game state machine** вҖ” phases:
  ```
  LOBBY вҶ’ DEALING вҶ’ AWAITING_PLAY вҶ’ AWAITING_DRAW_DECISION вҶ’
  AWAITING_COLOR_CHOICE вҶ’ AWAITING_UNO_CATCH вҶ’ ROUND_OVER вҶ’
  GAME_OVER
  ```

- [ ] **Move validation:**
  - Card matches discard top by color, number, or symbol
  - Wild cards always playable
  - Wild Draw Two / Wild Draw Color вҖ” technically only when no other match (challenge rule)

- [ ] **Action card effects:**
  - **Light side:** Skip, Reverse, Draw One, Wild, Wild Draw Two, Flip
  - **Dark side:** Skip Everyone, Reverse, Draw Five, Wild, Wild Draw Color, Flip
  - Direction tracking (clockwise/counter-clockwise)

- [ ] **The Flip mechanic:**
  - Flip all player hands (server tracks both sides)
  - Flip draw pile and discard pile
  - New discard top determines next match
  - Broadcast `FLIP_EVENT` to all clients for animation

- [ ] **UNO calling:**
  - Auto-detect when player has 1 card left
  - Window for calling UNO (before next player acts)
  - Other players can "catch" вҖ” penalty: draw 2

- [ ] **Scoring:**
  - Calculate points based on active side's values
  - Track cumulative score across rounds
  - Game ends at 500 points

- [ ] **Edge cases:**
  - Starting card is action вҶ’ apply to first player
  - Starting card is Wild Draw Two вҶ’ reshuffle
  - Starting card is Flip вҶ’ immediate flip to dark
  - 2-player mode: Reverse = Skip
  - Challenge rule for Wild Draw Two / Wild Draw Color

## Data Flow

```
Player taps card
  вҶ’ Client sends PLAY_CARD { cardId }
  вҶ’ Server validates move
  вҶ’ Server updates game state
  вҶ’ Server broadcasts to ALL:
      - CARD_PLAYED { card, playerId } (public info)
      - Updated opponent card counts
  вҶ’ Server sends to EACH player privately:
      - HAND_UPDATE { hand } (only their cards)
  вҶ’ If action card: server applies effect, broadcasts result
  вҶ’ If FLIP: server flips everything, sends FLIP_EVENT
  вҶ’ Server advances turn, broadcasts GAME_STATE
```

---

## Key Design Decisions

1. **Server-authoritative** вҖ” Server holds all game state, validates every move. Clients are "dumb" renderers. Prevents cheating.
2. **Private hands** вҖ” Each player only receives their own cards. Opponents see card counts only.
3. **Both sides tracked server-side** вҖ” Server always knows both sides of every card. Clients only see the active side of their hand (honor system enforced by the server only sending active side data).
4. **Optimistic UI** вҖ” Play card immediately on tap, revert if server rejects. On LAN the confirmation is <5ms so this is barely noticeable.
5. **No database** вҖ” All state in-memory. Games are ephemeral. Server restart = clean slate.
6. **No auth** вҖ” Players just enter a name. Player identity tied to Socket.IO session + a reconnect token stored in localStorage.
